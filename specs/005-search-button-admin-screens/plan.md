## Phase 0: Research

### Research Tasks

1. **Search Button Pattern**: Analyze existing button patterns in the codebase for consistent styling.

2. **Debounced Search Removal**: Identify all auto-search triggers to remove (useEffect with debounce).

3. **Enter Key Handling**: Implement onKeyDown handler for text inputs to trigger search on Enter.

### Research Findings

**Decision**: Use existing button styling from "新規ユーザー登録" button. Remove debounce useEffect, add explicit search handler.

**Rationale**:
- Consistent UI/UX across admin screens
- Explicit search provides better user control
- Enter key support maintains keyboard accessibility

**Alternatives Considered**:
- Keep auto-search + add button → Rejected: confusing UX with two search triggers
- Add debounce indicator → Rejected: adds complexity without clear value
