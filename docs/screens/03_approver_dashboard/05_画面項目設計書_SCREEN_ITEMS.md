# 画面項目設計書 (Screen Items Specification) - 最終承認者ダッシュボード

**対象画面:** 最終承認者ダッシュボード (Approver Dashboard)  
**バージョン:** 1.0  
**作成日:** 2026-06-12  
**ステータス:** 承認済み  

---

## 1. 画面レイアウト概要
画面中央に申請詳細、右側にこれまでの承認履歴タイムラインを配置し、下部に承認・却下アクション操作パネル（固定フッターレイアウト）を設ける。

---

## 3. ボタン・アクション定義 (Events & Actions)

1. **「承認する (Approve)」ボタン:**
   - 押下イベント: 最終確認ダイアログを表示。承認時にステータスを `APPROVED` に変更。経理部門に即座に通知を送信。
2. **「却下する (Reject)」ボタン:**
   - 押下イベント: `approver_comment` の入力文字数をバリデーション（10文字未満の場合は入力要求トーストを表示）。バリデーションクリア時、ステータスを `REJECTED_APPROVER` に変更して差し戻す。
3. **「履歴・タイムライン展開」リンク:**
   - タイムラインの詳細ログを展開し、接続IPアドレス等のセキュリティ監査情報を表示。
