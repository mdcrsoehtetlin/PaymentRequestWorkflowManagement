# 機能設計書 (Functional Specification) - 経理ダッシュボード

**対象画面:** 経理ダッシュボード (Accounting Dashboard)  
**バージョン:** 1.0  
**作成日:** 2026-06-12  
**ステータス:** 承認済み  

---

## 1. 機能概要 (Functional Overview)
最終承認された支払申請案件（ステータス：`Approved`）を対象に、経理（Accounting）ロールのユーザーが実際の支払処理（銀行振込・現金手渡し等）を行い、完了時にシステム上で「支払完了（Paid）」の登録を行うための画面である。

---

## 2. ユースケースと業務フロー (Usecases & Workflow)

### 2.1 主なユースケース
1. **承認済み支払待ちリストの表示:** 最終承認が完了（`Approved`）し、未払いの申請案件を一覧表示する。
2. **申請詳細・振込先情報の閲覧:** 支払金額、支払希望日、支払方法、口座情報を確認する。
3. **支店別支払アレルートの確認:** 申請者の所属支店に基づき、適切な支払ガイドラインを表示する（Mandalay支店の現金支払個別ルールなど）。
4. **支払完了（Paid）処理:** 実際の送金または現金支払いを完了した後、システム上で「支払完了」ボタンを押下してワークフローをクローズする。ステータスは `PAID` に遷移する。

### 2.2 業務フロー（経理支払処理）
```
[承認済み支払待ちリスト閲覧]
        │
    (申請選択)
        │
        ▼
[詳細画面の表示] ──► (支店チェック)
                            │
               ┌────────────┴────────────┐
               ▼ (Mandalay支店)          ▼ (他支店)
        【現金支払アラート】       【銀行振込アラート】
       「Toe San氏と要調整」     「標準銀行振込処理」
               │                         │
         (実際の支払実行)          (実際の支払実行)
               │                         │
               └────────────┬────────────┘
                            │
                  【支払完了ボタン押下】
                            │
                            ▼
                   [Paid (Completed)]
```

---

## 3. ビジネスルール (Business Rules)
1. **支店別支払アラートルール:**
   - 申請者の所属支店が **"Mandalay"** の場合、詳細画面に赤色太字で **「【重要】Mandalay支店：現金支払のため、Toe San氏と調整してください」** という警告メッセージを表示する。
   - 申請者の所属支店が **"Mandalay" 以外**（例：Yangon, Naypyidaw）の場合、青色で **「標準銀行振込処理」** と表示する。
2. **支払完了後の状態ロック:** ステータスが `PAID` に遷移した申請は、システム上の最終エンド状態（完了）となり、以降はいかなる状態遷移アクションも実行できず、完全な読み取り専用データとしてロックされる。

---

## 4. 権限制御 (Permission Control)
- **要求ロール:** `ACCOUNTING`
- **認証方式:** JWT Bearer トークン検証
- **認可ガード:** `RolesGuard` による経理ロールのアクセス制御。
