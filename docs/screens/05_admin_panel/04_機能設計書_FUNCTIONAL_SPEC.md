# 機能設計書 (Functional Specification) - システム管理者パネル

**対象画面:** システム管理者パネル (Admin Panel)  
**バージョン:** 1.0  
**作成日:** 2026-06-12  
**ステータス:** 承認済み  

---

## 1. 機能概要 (Functional Overview)
システム管理者（Admin）ロールのユーザーが、システム全体のユーザー管理（アカウント作成・編集・有効化）、マスタテーブルの設定確認、およびシステム全体で発生した全支払申請と承認ログ（監査ログ）の閲覧・検索を行うための管理コンソール画面である。

---

## 2. ユースケースと業務フロー (Usecases & Workflow)

### 2.1 主なユースケース
1. **ユーザーアカウント管理:** 新規ユーザーの登録（社員番号、氏名、メール、支店、ロールの割り当て）およびアカウントの有効化/無効化（`is_active`フラグ制御）の変更。
2. **システムマスタデータの閲覧:** ロール、ステータス、通貨、支払タイプ等の静的マスタ設定内容の確認。
3. **監査ログ（承認ログ）のグローバル閲覧:** 全支払申請で発生したタイムスタンプ履歴を横断的に検索・ダウンロードし、不正な状態操作やログインログがないかを監視する。

### 2.2 業務フロー（管理者アクション）
```
[管理者コンソールログイン] ──► [管理者メニュー]
                                     │
         ┌───────────────────────────┼───────────────────────────┐
         ▼                           ▼                           ▼
  【ユーザー追加・更新】      【マスタ設定確認】          【グローバル監査ログ検索】
  ・ロールの変更・無効化      ・lookupマスタのキャッシュ  ・全履歴のCSV出力
  ・パスワード再発行           ・Redisとの同期状況確認    ・IPアドレス監査
```

---

## 3. ビジネスルール (Business Rules)
1. **システム操作対象外ルール:** 管理者はシステム設定とアカウント管理を行う役割であり、自身がワークフロー（申請の起票、マネージャー確認、最終承認、支払処理）のトランザクションへ介入することは原則不可とする（監査上の利益相反防止）。
2. **不変ログ（不変性）の厳守:** 管理者であっても `approval_logs` の直接削除、レコードの編集は不可能。データベースレベルのトリガーにより保護される。

---

## 4. 権限制御 (Permission Control)
- **要求ロール:** `ADMIN`
- **認証方式:** JWT Bearer トークン検証
- **認可ガード:** `RolesGuard`による厳格な制限。
