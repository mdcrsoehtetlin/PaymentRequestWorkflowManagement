# 画面項目設計書 (Screen Items Specification) - 担当マネージャーダッシュボード

**対象画面:** 担当マネージャーダッシュボード (Manager Dashboard)  
**バージョン:** 2.0  
**作成日:** 2026-06-15  
**ステータス:** 承認済み  

---

## 1. 画面レイアウト概要
マネージャー画面は、上部に「確認待ち申請キュー一覧」、下部に選択された申請の詳細（明細グリッド、添付領収書プレビュー、差し戻しコメント入力フォーム）を表示する分割ビューレイアウトで構成する。

---

## 3. ボタン・アクション定義 (Events & Actions)

1. **「確認完了 (Verify / OK)」ボタン:**
   - 押下イベント: コメントがある場合は任意で付与し、ステータスを `MANAGER_VERIFIED` (4) に変更。申請の割当 (`current_assigned_to_user_id`) を申請者に戻して登録する。申請完了トーストを表示し、キュー一覧を更新する。
2. **「却下する (Reject)」ボタン:**
   - 押下イベント: `manager_comment` の文字数をチェック（10文字未満の場合はエラーメッセージを表示して処理中断）。バリデーションクリア後、ステータスを `REJECTED_MANAGER` (5) に変更し、申請の割当を申請者に戻して差し戻す。
3. **「領収書ダウンロード」リンク:**
   - 押下イベント: サーバーから領収書ファイルを安全に取得し、ローカルにダウンロード保存する。
4. **「自動レビュー開始」イベント (Auto Review Transition):**
   - 発生イベント: マネージャーがキュー一覧からステータス `SUBMITTED_MANAGER` (2) の申請詳細を選択して開いた時。
   - 処理内容: ステータスを `MANAGER_REVIEWING` (3) に自動更新し、承認履歴ログに「Manager opened request and started reviewing」を登録し、WebSocketで関係者へステータス変更を通知する。
