# 画面項目設計書 (Screen Items Specification) - 申請者ダッシュボード

**対象画面:** 申請者ダッシュボード (Applicant Dashboard)  
**バージョン:** 1.0  
**作成日:** 2026-06-12  
**ステータス:** 承認済み  

---

## 1. 画面レイアウト概要
本画面は、左側に申請中のステータスを表示する「申請履歴リスト・タイムライン」、右側に詳細情報および「新規支払申請・編集フォーム」を配置する2カラムレスポンシブレイアウトで構成する。

---

## 3. ボタン・アクション定義 (Events & Actions)

1. **「一時保存 (Save Draft)」ボタン:**
   - 押下イベント: フォームの入力内容を検証し、未入力項目があってもエラーとせず、ステータスを `DRAFT` としてデータベースへ保存する。
2. **「マネージャーへ提出 (Submit to Manager)」ボタン:**
   - 押下イベント: 全必須項目の入力チェックおよび領収書ファイル添付チェックを実施。バリデーション成功時、ステータスを `SUBMITTED_MANAGER` に変更して送信する。
3. **「最終承認者へ送信 (Submit to Approver)」ボタン:**
   - 押下イベント: ステータスが `MANAGER_VERIFIED` の場合のみ活性化。最終承認者へ申請を回送し、ステータスを `SUBMITTED_APPROVER` へ変更する。
4. **「下書きを削除 (Delete Draft)」ボタン:**
   - 押下イベント: ステータスが `DRAFT` の場合のみ表示。論理削除確認ダイアログを表示し、承認時に `is_deleted = TRUE` を更新してダッシュボード一覧へ戻る。
